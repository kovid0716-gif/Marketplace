# Marketplace API Backend

Профессиональный бэкенд для платформы маркетплейса, реализованный на фреймворке **NestJS** с использованием ролевой модели доступа (RBAC), контейнеризацией через **Docker** и транзакционной работой с базой данных через **Prisma ORM**.

## 🛠 Технологический стек

* **Framework:** NestJS (TypeScript)
* **Database:** PostgreSQL
* **ORM:** Prisma
* **Authentication:** JWT (JsonWebToken) & Passport.js
* **Containerization:** Docker / Docker Compose
* **API Testing & Docs:** Postman, Swagger (OpenAPI 3.0)

---

## 🚀 Функционал приложения

1. **Аутентификация и авторизация (Автоматический JWT):**
   * Регистрация пользователей с разделением ролей: Покупатель (`USER`) и Продавец (`SELLER`).
   * Защита эндпоинтов с помощью JWT-стратегии и кастомных гвардов (`JwtAuthGuard`, `RolesGuard`).
2. **Управление товарами (Products CRUD):**
   * Просмотр товаров доступен всем пользователям.
   * Создание товаров доступно исключительно пользователям с ролью `SELLER`.
   * Редактирование и удаление товара разрешено только его непосредственному владельцу (`ownerId`).
3. **Модуль заказов (Orders):**
   * Оформление заказов покупателями.
   * Автоматический расчет стоимости на стороне бэкенда в рамках единой базы данных.
   * Просмотр истории личных заказов.

## 📂 Структура проекта

Основные папки и модули приложения:
```text
├── prisma/                  # Схема базы данных и миграции
│   ├── migrations/          # SQL-файлы миграций базы данных
│   └── schema.prisma        # Определение моделей (User, Product, Order)
├── src/                     # Исходный код приложения
│   ├── auth/                # Модуль аутентификации (логика, стратегии, гварды)
│   ├── products/            # Модуль управления товарами (CRUD)
│   ├── orders/              # Модуль оформления и истории заказов
│   ├── prisma/              # Глобальный сервис для работы с Prisma Client
│   ├── main.ts              # Точка входа приложения
│   └── app.module.ts        # Корневой модуль
├── docker-compose.yml       # Конфигурация Docker контейнеров (App + DB)
├── Dockerfile.txt           # Инструкция сборки Docker-образа приложения
└── package.json             # Зависимости и скрипты проекта
